# 3. Publisher와 Subscriber

## Publisher

`class_pub.py`는 `/message` 토픽으로 `String` 메시지를 발행한다.

```python
self.pub = self.create_publisher(String, "message", 10)
self.create_timer(1.0, self.timer_callback)
```

```python
def timer_callback(self):
    msg = String()
    msg.data = f"첫번째 프로그램입니다. {self.count}"
    self.pub.publish(msg)
```

## Subscriber

`class_sub.py`는 같은 토픽을 구독한다.

```python
self.create_subscription(
    String,
    "message",
    self.sub_callback,
    10
)
```

```python
def sub_callback(self, msg):
    self.get_logger().info(msg.data)
```

실행:

```bash
ros2 run khj_basic class_pub
ros2 run khj_basic class_sub
```

확인:

```bash
ros2 topic echo /message
ros2 topic info /message
```

# 4. Header와 QoS

## Header 메시지

`header_pub.py`는 현재 시간과 `frame_id`를 `/time` 토픽으로 발행한다.

```python
msg = Header()
msg.frame_id = "time test"
msg.stamp = self.get_clock().now().to_msg()
self.pub.publish(msg)
```

```bash
ros2 run khj_basic header_pub
ros2 topic echo /time
```

## QoS 설정

`qos_test_pub.py`와 `qos_test_sub.py`에서 다음 QoS를 사용하였다.

```python
qos_profile = QoSProfile(
    history=QoSHistoryPolicy.KEEP_ALL,
    reliability=QoSReliabilityPolicy.RELIABLE,
    durability=QoSDurabilityPolicy.TRANSIENT_LOCAL,
)
```

| 설정 | 의미 |
|---|---|
| `KEEP_ALL` | 메시지를 모두 보관 |
| `RELIABLE` | 신뢰성 있는 전달 우선 |
| `TRANSIENT_LOCAL` | 늦게 실행된 Subscriber도 이전 메시지 수신 |

실행:

```bash
ros2 run khj_basic qos_test_pub
ros2 run khj_basic qos_test_sub
```

QoS 확인:

```bash
ros2 topic info /message --verbose
```

# 8. Service Server와 Client

## Service Server

`service_server.py`는 두 정수를 더하고 결과가 홀수인지 판별한다.

```python
self.create_service(
    AddAndOdd,
    "add_server",
    self.add_callback
)
```

```python
response.sum = request.inta + request.intb

if response.sum % 2:
    response.odd = "Two ints sum is odd"
else:
    response.odd = "Two ints sum is not odd"
```

현재 Server는 콜백 안에서 10초 동안 대기한다.

```python
time.sleep(10)
```

## 비동기 Service Client

`service_client.py`는 3초마다 요청을 보낸다.

```python
self.future = self.client.call_async(self.request)
self.future.add_done_callback(self.done_callback)
```

`call_async()`를 사용하므로 결과가 올 때까지 프로그램 전체를 멈추지 않는다.

실행:

```bash
ros2 run khj_basic service_server
ros2 run khj_basic service_client
```

서비스 확인:

```bash
ros2 service list
ros2 service type /add_server
```

터미널에서 직접 요청:

```bash
ros2 service call /add_server \
user_interface/srv/AddAndOdd \
"{inta: 4, intb: 8}"
```

# 9. 멀티스레드 Service Server

`service_thread_server.py`에서는 다음 기능을 사용하였다.

```python
from rclpy.callback_groups import ReentrantCallbackGroup
from rclpy.executors import MultiThreadedExecutor
```

Callback Group 설정:

```python
self.callback_group = ReentrantCallbackGroup()
```

멀티스레드 Executor 설정:

```python
executor = MultiThreadedExecutor(num_threads=4)
executor.add_node(node)
executor.spin()
```

여러 스레드가 같은 데이터를 동시에 변경하는 Race Condition을 막기 위해 Lock을 사용하였다.

```python
self.lock = threading.Lock()

with self.lock:
    response.sum = request.inta + request.intb
```

실행:

```bash
ros2 run khj_basic service_thread_server
ros2 run khj_basic service_client
```

# 10. Parameter 실습

## Parameter 선언

`my_param.py`에서 문자열 Parameter를 선언하였다.

```python
self.declare_parameter(
    "my_param",
    "내가 만든 클래스 노드 안의 파라미터"
)
```

값 읽기:

```python
self.my_param = (
    self.get_parameter("my_param")
    .get_parameter_value()
    .string_value
)
```

Parameter가 변경되면 Callback에서 값을 갱신한다.

```python
self.add_on_set_parameters_callback(
    self.parameter_callback
)
```

실행:

```bash
ros2 run khj_basic my_param
```

확인 및 변경:

```bash
ros2 param list /tparam
ros2 param get /tparam my_param
ros2 param set /tparam my_param "변경한 값"
```

# 11. 비동기 Parameter Client

`param_async.py`는 `/tparam` 노드의 Parameter를 외부에서 변경한다.

```python
self.parameter_client = AsyncParameterClient(
    self,
    "/tparam"
)
```

1초마다 새로운 문자열 값 전송:

```python
parameter = Parameter(
    name="my_param",
    type_=Parameter.Type.STRING,
    value=f"외부 노드에서 변경한 값 {self.count}"
)

future = self.parameter_client.set_parameters(
    [parameter]
)
```

실행:

```bash
ros2 run khj_basic my_param
ros2 run khj_basic param_async
```

# 14. 실습 핵심 정리

1. `setup.py`에 등록한 Python 파일은 `ros2 run`으로 실행할 수 있다.
2. Topic은 지속적으로 데이터를 발행하고 구독할 때 사용한다.
3. QoS로 메시지 보관, 신뢰성 및 지속성을 설정할 수 있다.
4. Launch 파일은 여러 노드를 한 번에 실행한다.
5. `.msg`와 `.srv` 파일로 사용자 정의 Interface를 만들 수 있다.
6. Service는 Request와 Response 방식으로 동작한다.
7. `call_async()`는 응답을 기다리는 동안 다른 작업을 계속 처리한다.
8. `MultiThreadedExecutor`는 여러 Callback을 여러 스레드에서 처리한다.
9. Parameter는 실행 중인 노드의 설정값을 외부에서 확인하거나 변경할 수 있다.
10. Parameter 값은 YAML 파일로 저장하고 다시 불러올 수 있다.
