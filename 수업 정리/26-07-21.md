# 1. ROS 2 파일 시스템

현재 실습 워크스페이스는 다음 위치에 있다.

```text
~/2026_manipulator/khj_ws
```

기본 구조는 다음과 같다.

```text
khj_ws/
├── src/       # 직접 작성하거나 내려받은 패키지
├── build/     # 빌드 중간 결과
├── install/   # 실행에 필요한 설치 결과와 환경 설정
└── log/       # 빌드 로그
```

`src` 폴더에는 직접 작성하는 ROS 2 패키지를 넣는다.

```text
khj_ws/src/khj_basic
```

> `build`, `install`, `log` 폴더는 `colcon build`를 실행하면 자동으로 만들어진다.

# 2. Python ROS 2 패키지 생성

## 8.1 워크스페이스의 `src` 폴더로 이동

```bash
cd ~/2026_manipulator/khj_ws/src
```

## 8.2 패키지 생성

```bash
ros2 pkg create --build-type ament_python khj_basic
```

### 명령어 의미

| 항목 | 의미 |
|---|---|
| `ros2 pkg create` | 새로운 ROS 2 패키지 생성 |
| `--build-type ament_python` | Python 패키지 형식 사용 |
| `khj_basic` | 생성할 패키지 이름 |

> 현재 `khj_basic` 패키지가 이미 존재하므로 이 명령을 다시 실행할 필요는 없다.

# 3. `khj_basic` 패키지 구조

현재 패키지의 핵심 구조는 다음과 같다.

```text
khj_basic/
├── khj_basic/
│   ├── __init__.py
│   └── simple_pub.py
├── resource/
│   └── khj_basic
├── test/
├── package.xml
├── setup.cfg
└── setup.py
```

## 9.1 파일별 역할

| 파일 또는 폴더 | 역할 |
|---|---|
| `khj_basic/` | Python 소스 코드가 들어가는 내부 모듈 폴더 |
| `__init__.py` | 해당 폴더를 Python 패키지로 인식시킴 |
| `simple_pub.py` | 이번 실습에서 작성한 Python 실행 파일 |
| `resource/khj_basic` | ament 인덱스가 패키지를 찾을 때 사용하는 파일 |
| `test/` | 코드 검사와 테스트 파일이 들어가는 폴더 |
| `package.xml` | 패키지 이름, 버전, 관리자, 라이선스, 의존성 정보 |
| `setup.py` | Python 패키지 설치 및 실행 파일 등록 설정 |
| `setup.cfg` | ROS 2 실행 파일의 설치 위치 설정 |

GitHub에서 패키지 폴더로 바로 이동하려면 다음 상대 링크를 사용할 수 있다.

- [`khj_basic` 패키지](../khj_ws/src/khj_basic)
- [`simple_pub.py`](../khj_ws/src/khj_basic/khj_basic/simple_pub.py)

# 5. `setup.py`에 실행 파일 등록

Python 파일을 `ros2 run`으로 실행하려면 `setup.py`의 `console_scripts`에 등록해야 한다.

```python
entry_points={
    'console_scripts': [
        'simple_pub = khj_basic.simple_pub:main',
    ],
},
```

## 11.1 등록 형식

```text
실행 파일 이름 = Python 모듈 경로:실행 함수
```

현재 설정을 분리하면 다음과 같다.

| 부분 | 값 | 의미 |
|---|---|---|
| 실행 파일 이름 | `simple_pub` | `ros2 run`에서 사용할 이름 |
| 패키지 모듈 | `khj_basic` | 내부 Python 패키지 폴더 |
| Python 파일 | `simple_pub` | `simple_pub.py` |
| 실행 함수 | `main` | 파일 안의 `main()` 함수 |

따라서 다음 명령으로 실행할 수 있다.

```bash
ros2 run khj_basic simple_pub
```

# 6. `package.xml`

`package.xml`에는 패키지에 대한 기본 정보가 들어 있다.

현재 주요 내용:

```xml
<name>khj_basic</name>
<version>0.0.0</version>
<maintainer email="khjoo990408@gmail.com">khjoo</maintainer>

<export>
  <build_type>ament_python</build_type>
</export>
```

### 주요 태그

| 태그 | 의미 |
|---|---|
| `<name>` | 패키지 이름 |
| `<version>` | 패키지 버전 |
| `<description>` | 패키지 설명 |
| `<maintainer>` | 패키지 관리자 |
| `<license>` | 라이선스 |
| `<depend>` | 빌드와 실행에 필요한 의존성 |
| `<test_depend>` | 테스트에 필요한 의존성 |
| `<build_type>` | 패키지 빌드 형식 |

현재 `simple_pub.py`는 `rclpy`를 사용하지 않으므로 `rclpy` 의존성이 등록되어 있지 않다.

나중에 실제 ROS 2 Python 노드를 작성한다면 다음과 같은 의존성이 필요하다.

```xml
<exec_depend>rclpy</exec_depend>
```

또는 패키지 생성 단계에서 다음처럼 지정할 수 있다.

```bash
ros2 pkg create \
  --build-type ament_python \
  --dependencies rclpy \
  패키지이름
```

# 7. `setup.cfg`

현재 `setup.cfg`의 내용은 다음과 같다.

```ini
[develop]
script_dir=$base/lib/khj_basic

[install]
install_scripts=$base/lib/khj_basic
```

이 설정은 Python 실행 파일을 다음 형식의 위치에 설치하도록 지정한다.

```text
install/khj_basic/lib/khj_basic/
```

ROS 2는 이 설치 위치를 이용하여 `ros2 run khj_basic simple_pub` 명령으로 실행 파일을 찾는다.

# 8. 패키지 빌드

## 14.1 워크스페이스 루트로 이동

```bash
cd ~/2026_manipulator/khj_ws
```

> 빌드는 `src` 폴더나 `khj_ws/khj_ws` 경로가 아니라 워크스페이스 루트인 `khj_ws`에서 실행한다.

## 14.2 특정 패키지만 빌드

```bash
colcon build --symlink-install --packages-select khj_basic
```

### 옵션 설명

| 옵션 | 의미 |
|---|---|
| `colcon build` | 워크스페이스의 패키지 빌드 |
| `--symlink-install` | Python 소스 수정 내용을 다시 설치하지 않고 빠르게 반영 |
| `--packages-select khj_basic` | `khj_basic` 패키지만 선택하여 빌드 |

전체 패키지를 빌드하려면 다음과 같이 실행한다.

```bash
colcon build --symlink-install
```

# 12. 현재 `simple_pub.py`는 실제 ROS 노드인가?

현재 코드:

```python
def main():
    print("첫번째 프로그램입니다.")
```

이 코드는 ROS 2 패키지 안에 들어 있고 `ros2 run`으로 실행할 수 있지만, 아직 다음 기능을 사용하지 않는다.

- `rclpy.init()`
- `Node` 클래스
- Publisher
- Subscriber
- Service
- Action
- Parameter
- `rclpy.spin()`

따라서 현재 프로그램은 엄밀히 말하면 다음과 같다.

> ROS 2 패키지에 등록된 Python 실행 파일이지만, 아직 ROS 그래프에 등록되는 실제 ROS 2 노드는 아니다.

현재 프로그램을 실행해도 다음 명령에 노드 이름이 나타나지 않는다.

```bash
ros2 node list
```

# 13. 실제 ROS 2 Python 노드의 기본 형태

실제 ROS 2 노드는 대략 다음 구조를 가진다.

```python
import rclpy
from rclpy.node import Node


class SimpleNode(Node):

    def __init__(self):
        super().__init__('simple_node')
        self.get_logger().info('Simple node started.')


def main(args=None):
    rclpy.init(args=args)

    node = SimpleNode()
    rclpy.spin(node)

    node.destroy_node()
    rclpy.shutdown()


if __name__ == '__main__':
    main()
```

이 코드에서는 다음 과정이 수행된다.

```text
ROS 2 초기화
    ↓
노드 생성
    ↓
Executor에서 콜백 대기 및 처리
    ↓
노드 종료
    ↓
ROS 2 종료
```

# 14. `.gitignore`에 제외할 항목

Python을 실행하면 `__pycache__` 폴더와 `.pyc` 파일이 자동으로 만들어질 수 있다.

이 파일은 소스 코드가 아니므로 일반적으로 GitHub에 올리지 않는다.

프로젝트 최상위 `.gitignore`에 다음 내용을 추가할 수 있다.

```gitignore
# Python cache
__pycache__/
*.pyc

# ROS 2 build results
build/
install/
log/
```
