# 5. `setup.py`에 실행 파일 등록

Python 파일을 `ros2 run`으로 실행하려면 `setup.py`의 `console_scripts`에 등록해야 한다.

```python
entry_points={
    'console_scripts': [
        'simple_pub = khj_basic.simple_pub:main',
    ],
},
```

## 11.1 등록 형식

```text
실행 파일 이름 = Python 모듈 경로:실행 함수
```

현재 설정을 분리하면 다음과 같다.

| 부분 | 값 | 의미 |
|---|---|---|
| 실행 파일 이름 | `simple_pub` | `ros2 run`에서 사용할 이름 |
| 패키지 모듈 | `khj_basic` | 내부 Python 패키지 폴더 |
| Python 파일 | `simple_pub` | `simple_pub.py` |
| 실행 함수 | `main` | 파일 안의 `main()` 함수 |

따라서 다음 명령으로 실행할 수 있다.

```bash
ros2 run khj_basic simple_pub
```

# 7. `setup.cfg`와 `setup.py`

## 7.1 `setup.cfg`

`setup.cfg`는 Python 실행 파일이 설치될 위치를 지정하는 파일이다.

현재 내용은 다음과 같다.

```ini
[develop]
script_dir=$base/lib/khj_basic

[install]
install_scripts=$base/lib/khj_basic
```

실행 파일은 다음 경로에 설치된다.

```text
install/khj_basic/lib/khj_basic/
```

ROS 2는 이 경로에서 실행 파일을 찾아 `ros2 run` 명령으로 실행한다.

---

## 7.2 `setup.py`

Python 파일을 `ros2 run`으로 실행하려면 `setup.py`의 `console_scripts`에 등록해야 한다.

현재 등록된 실행 파일은 다음과 같다.

```python
entry_points={
    'console_scripts': [
        "simple_pub = khj_basic.simple_pub:main",
        "class_pub = khj_basic.class_pub:main",
        "class_sub = khj_basic.class_sub:main"
    ],
},
```

### 등록 형식

```text
실행 파일 이름 = Python 모듈 경로:실행 함수
```

각 항목의 의미는 다음과 같다.

| 실행 파일 이름 | Python 파일 | 실행 함수 |
|---|---|---|
| `simple_pub` | `khj_basic/simple_pub.py` | `main()` |
| `class_pub` | `khj_basic/class_pub.py` | `main()` |
| `class_sub` | `khj_basic/class_sub.py` | `main()` |

따라서 다음과 같이 실행한다.

```bash
ros2 run khj_basic simple_pub
ros2 run khj_basic class_pub
ros2 run khj_basic class_sub
```

> 실행 파일을 새로 추가하거나 `setup.py`를 수정한 경우에는 반드시 패키지를 다시 빌드해야 한다.

# 10. 클래스 기반 Publisher 노드 작성

파일 위치:

```text
~/2026_manipulator/khj_ws/src/khj_basic/khj_basic/class_pub.py
```

Publisher는 `message` 토픽으로 문자열 메시지를 발행한다.

```python
import rclpy
from rclpy.node import Node
from std_msgs.msg import String


class M_pub(Node):

    def __init__(self):
        super().__init__('message_pub')

        # 1초마다 timer_callback 함수를 실행
        self.create_timer(1.0, self.timer_callback)

        # String 메시지를 message 토픽으로 발행하는 Publisher 생성
        self.pub = self.create_publisher(
            String,
            'message',
            10
        )

        self.count = 0

    def timer_callback(self):
        msg = String()
        msg.data = f'첫번째 프로그램 {self.count}'

        self.get_logger().info(msg.data)
        self.pub.publish(msg)

        self.count += 1


def main(args=None):
    rclpy.init(args=args)

    node = M_pub()

    try:
        rclpy.spin(node)

    except KeyboardInterrupt:
        node.get_logger().info('KeyboardInterrupt')

    finally:
        node.destroy_node()
        rclpy.shutdown()


if __name__ == '__main__':
    main()
```

---

### ROS 2 라이브러리 불러오기

| `String` | 문자열 데이터를 전송하기 위한 메시지 자료형 |

# 11. 클래스 기반 Subscriber 노드 작성

파일 위치:

```text
~/2026_manipulator/khj_ws/src/khj_basic/khj_basic/class_sub.py
```

Subscriber는 Publisher가 `/message` 토픽으로 발행한 문자열을 수신한다.

```python
import rclpy
from rclpy.node import Node
from std_msgs.msg import String


class M_sub(Node):

    def __init__(self):
        super().__init__('message_sub')

        # message 토픽을 구독하는 Subscription 생성
        self.create_subscription(
            String,
            'message',
            self.sub_callback,
            10
        )

    def sub_callback(self, msg: String):
        self.get_logger().info(msg.data)


def main(args=None):
    rclpy.init(args=args)

    node = M_sub()

    try:
        rclpy.spin(node)

    except KeyboardInterrupt:
        node.get_logger().info('KeyboardInterrupt')

    finally:
        node.destroy_node()
        rclpy.shutdown()


if __name__ == '__main__':
    main()
```

> 기존 실습 코드에는 Subscriber 클래스 이름과 노드 이름이 `M_pub`, `message_pub`으로 작성되어 있었다. Publisher와 구분하기 쉽도록 `M_sub`, `message_sub`로 정리하였다.

---

## 11.1 Subscription 생성

```python
self.create_subscription(
    String,
    'message',
    self.sub_callback,
    10
)
```

| 인수 | 의미 |
|---|---|
| `String` | 수신할 메시지 자료형 |
| `message` | 구독할 토픽 이름 |
| `self.sub_callback` | 메시지 수신 시 실행할 함수 |
| `10` | QoS 메시지 대기열 크기 |

Publisher와 Subscriber의 메시지 자료형과 토픽 이름이 같아야 통신할 수 있다.

```text
Publisher                     Subscriber
String                        String
/message      ─────────▶      /message
```

---

## 11.2 Subscriber 콜백 함수

```python
def sub_callback(self, msg: String):
    self.get_logger().info(msg.data)
```

Publisher가 메시지를 발행하면 `sub_callback()` 함수가 실행된다.

수신한 `String` 메시지의 실제 문자열은 다음 필드에 들어 있다.

```python
msg.data
```

# 12. Publisher와 Subscriber 통신 테스트

Publisher와 Subscriber는 서로 다른 터미널에서 실행한다.

## 12.1 첫 번째 터미널: Publisher 실행

```bash
ros2 run khj_basic class_pub
```

예상 출력:

```text
[INFO] [message_pub]: 첫번째 프로그램 0
[INFO] [message_pub]: 첫번째 프로그램 1
[INFO] [message_pub]: 첫번째 프로그램 2
```

Publisher는 1초마다 `/message` 토픽으로 문자열을 발행한다.

---

## 12.2 두 번째 터미널: Subscriber 실행

```bash
ros2 run khj_basic class_sub
```

예상 출력:

```text
[INFO] [message_sub]: 첫번째 프로그램 0
[INFO] [message_sub]: 첫번째 프로그램 1
[INFO] [message_sub]: 첫번째 프로그램 2
```

Subscriber는 `/message` 토픽을 구독하여 수신한 문자열을 출력한다.

---

## 12.3 통신 구조

```text
/message_pub
     │
     │ std_msgs/msg/String
     ▼
  /message
     │
     ▼
/message_sub
```

- Publisher 노드: `/message_pub`
- Subscriber 노드: `/message_sub`
- 토픽 이름: `/message`
- 메시지 자료형: `std_msgs/msg/String`
- 발행 주기: 1초

# 18. `String` Publisher 노드 작성

파일 위치:

```text
~/2026_manipulator/khj_ws/src/khj_basic/khj_basic/class_pub.py
```

Publisher 노드는 `std_msgs/msg/String` 자료형의 메시지를 발행한다.

```python
import rclpy
from rclpy.node import Node
from std_msgs.msg import String


class M_pub(Node):
    def __init__(self):
        super().__init__("message_pub")

        self.create_timer(1, self.timer_callback)

        self.pub = self.create_publisher(
            String,
            "message",
            10,
        )

        self.count = 0

    def timer_callback(self):
        msg = String()
        msg.data = f"첫번째 프로그램 {self.count}"

        self.get_logger().info(msg.data)
        self.pub.publish(msg)

        self.count += 1


def main(args=None):
    rclpy.init(args=args)

    node = M_pub()
    rclpy.spin(node)

    node.destroy_node()
    rclpy.shutdown()


if __name__ == "__main__":
    main()
```

## 18.1 Publisher 생성

```python
self.pub = self.create_publisher(
    String,
    "message",
    10,
)
```

각 인자의 의미는 다음과 같다.

| 인자 | 값 | 의미 |
|---|---|---|
| 메시지 자료형 | `String` | 전송할 메시지의 형식 |
| 토픽 이름 | `"message"` | 메시지를 발행할 토픽 |
| 큐 크기 | `10` | 처리하지 못한 메시지를 보관할 수 있는 크기 |

실제 토픽 이름은 다음과 같이 표시된다.

```text
/message
```

## 18.2 Timer 생성

```python
self.create_timer(1, self.timer_callback)
```

1초마다 `timer_callback()` 함수를 실행한다.

```text
1초 경과
   ↓
timer_callback() 호출
   ↓
메시지 생성
   ↓
메시지 발행
```

## 18.3 메시지 객체 생성

```python
msg = String()
msg.data = f"첫번째 프로그램 {self.count}"
```

`String` 메시지는 실제 문자열을 `data` 필드에 저장한다.

예상되는 메시지는 다음과 같다.

```text
첫번째 프로그램 0
첫번째 프로그램 1
첫번째 프로그램 2
```

## 18.4 메시지 발행

```python
self.pub.publish(msg)
```

생성한 메시지를 `/message` 토픽으로 발행한다.

## 18.5 Logger 사용

```python
self.get_logger().info(msg.data)
```

노드의 실행 상태나 메시지 내용을 터미널에 출력한다.

`print()`와 달리 ROS 2의 노드 이름, 로그 등급 등의 정보가 함께 표시된다.

# 19. `String` Subscriber 노드 작성

파일 위치:

```text
~/2026_manipulator/khj_ws/src/khj_basic/khj_basic/class_sub.py
```

Subscriber 노드는 `/message` 토픽으로 발행된 `String` 메시지를 수신한다.

```python
import rclpy
from rclpy.node import Node
from std_msgs.msg import String


class M_sub(Node):
    def __init__(self):
        super().__init__("message_sub")

        self.create_subscription(
            String,
            "message",
            self.sub_callback,
            10,
        )

    def sub_callback(self, msg):
        self.get_logger().info(msg.data)


def main(args=None):
    rclpy.init(args=args)

    node = M_sub()
    rclpy.spin(node)

    node.destroy_node()
    rclpy.shutdown()


if __name__ == "__main__":
    main()
```

## 19.1 Subscriber 생성

```python
self.create_subscription(
    String,
    "message",
    self.sub_callback,
    10,
)
```

각 인자의 의미는 다음과 같다.

| 인자 | 값 | 의미 |
|---|---|---|
| 메시지 자료형 | `String` | 수신할 메시지 자료형 |
| 토픽 이름 | `"message"` | 구독할 토픽 |
| Callback | `self.sub_callback` | 메시지 도착 시 실행할 함수 |
| 큐 크기 | `10` | 수신 메시지 보관 크기 |

Publisher와 Subscriber가 통신하려면 다음 두 조건이 같아야 한다.

```text
토픽 이름이 같아야 함
메시지 자료형이 같아야 함
```

현재 Publisher와 Subscriber는 모두 다음 설정을 사용한다.

```text
토픽: /message
자료형: std_msgs/msg/String
```

## 19.2 Subscriber Callback

```python
def sub_callback(self, msg):
    self.get_logger().info(msg.data)
```

새로운 메시지가 도착할 때마다 자동으로 실행된다.

수신된 `String` 메시지의 문자열은 다음과 같이 읽는다.

```python
msg.data
```

# 20. Publisher와 Subscriber의 통신 구조

현재 통신 구조는 다음과 같다.

```text
/message_pub
     │
     │ /message
     │ std_msgs/msg/String
     ▼
/message_sub
```

Publisher가 메시지를 발행하면 Subscriber의 Callback 함수가 실행된다.

```text
Publisher Timer 실행
        ↓
String 메시지 생성
        ↓
/message 토픽으로 발행
        ↓
Subscriber가 메시지 수신
        ↓
sub_callback() 실행
        ↓
수신 내용 로깅
```

Publisher와 Subscriber의 노드 이름을 동일하게 지정하면 `rqt_graph`에서 하나의 노드처럼 겹쳐 보일 수 있다.

따라서 다음과 같이 서로 다른 노드 이름을 사용하는 것이 좋다.

```python
# Publisher
super().__init__("message_pub")

# Subscriber
super().__init__("message_sub")
```

# 21. `Header` 메시지를 이용한 시간 발행

파일 위치:

```text
~/2026_manipulator/khj_ws/src/khj_basic/khj_basic/header_pub.py
```

`Header` 메시지에는 시간 정보와 기준 좌표계 또는 데이터 구분용 문자열을 저장할 수 있다.

```python
from std_msgs.msg import Header
```

주요 필드는 다음과 같다.

| 필드 | 역할 |
|---|---|
| `stamp` | 메시지가 생성된 시간 |
| `stamp.sec` | 초 단위 시간 |
| `stamp.nanosec` | 나노초 단위 시간 |
| `frame_id` | 기준 좌표계 또는 메시지 구분 문자열 |

작성한 노드는 다음과 같다.

```python
import rclpy
from rclpy.node import Node
from std_msgs.msg import Header


class Header_pub(Node):
    def __init__(self):
        super().__init__("header_pub")

        self.create_timer(0.1, self.timer_callback)

        self.pub = self.create_publisher(
            Header,
            "time",
            10,
        )

    def timer_callback(self):
        msg = Header()

        msg.frame_id = "time test"
        msg.stamp = self.get_clock().now().to_msg()

        self.pub.publish(msg)

        self.get_logger().info(
            f"frame_id: {msg.frame_id}, "
            f"time: {msg.stamp.sec}.{msg.stamp.nanosec}"
        )


def main(args=None):
    rclpy.init(args=args)

    node = Header_pub()
    rclpy.spin(node)

    node.destroy_node()
    rclpy.shutdown()


if __name__ == "__main__":
    main()
```

## 21.1 10 Hz Timer

```python
self.create_timer(0.1, self.timer_callback)
```

0.1초마다 Callback 함수가 실행된다.

```text
1초 ÷ 0.1초 = 10회
```

따라서 발행 주파수는 10 Hz이다.

다음처럼 작성해도 같은 의미이다.

```python
self.create_timer(1 / 10, self.timer_callback)
```

## 21.2 현재 ROS 시간 가져오기

```python
msg.stamp = self.get_clock().now().to_msg()
```

현재 노드의 Clock에서 시간을 가져온 후 `Header` 메시지에 저장할 수 있는 형식으로 변환한다.

## 21.3 `/time` 토픽

```python
self.pub = self.create_publisher(
    Header,
    "time",
    10,
)
```

토픽과 자료형은 다음과 같다.

```text
토픽 이름: /time
메시지 자료형: std_msgs/msg/Header
```

토픽 내용을 직접 확인하려면 다음 명령을 실행한다.

```bash
ros2 topic echo /time
```

# 30. 예제의 전체 노드 및 토픽 구성

| Publisher 노드 | 토픽 | 메시지 자료형 | Subscriber 노드 |
|---|---|---|---|
| `/m_pub` | `/message1` | `std_msgs/msg/String` | `/m_sub`, `/mt_sub` |
| `/m_pub` | `/message2` | `std_msgs/msg/String` | `/m2_sub` |
| `/t_pub` | `/time` | `std_msgs/msg/Header` | `/mt_sub` |

전체 연결 구조는 다음과 같다.

```text
                         ┌── /message1 ──> /m_sub
                         │
/m_pub ──────────────────┤
                         │
                         └── /message1 ──> /mt_sub
                                               ▲
                                               │
/t_pub ──────────────── /time ─────────────────┘


/m_pub ────────────── /message2 ───────────> /m2_sub
```

하나의 토픽은 여러 Subscriber가 동시에 구독할 수 있다.

따라서 `/message1` 토픽으로 한 번 발행된 메시지를 `/m_sub`와 `/mt_sub`가 각각 수신한다.

# 31. 두 개의 토픽을 발행하는 `m_pub`

파일 위치:

```text
khj_example_1/khj_example_1/m_pub.py
```

```python
import rclpy
from rclpy.node import Node
from std_msgs.msg import String


class Message_pub(Node):
    def __init__(self):
        super().__init__("m_pub")

        self.create_timer(1, self.timer_callback)

        self.pub1 = self.create_publisher(
            String,
            "message1",
            10,
        )

        self.pub2 = self.create_publisher(
            String,
            "message2",
            10,
        )

    def timer_callback(self):
        msg1 = String()
        msg1.data = "message1"
        self.pub1.publish(msg1)

        msg2 = String()
        msg2.data = "message2"
        self.pub2.publish(msg2)


def main(args=None):
    rclpy.init(args=args)

    node = Message_pub()
    rclpy.spin(node)

    node.destroy_node()
    rclpy.shutdown()


if __name__ == "__main__":
    main()
```

하나의 노드 안에 Publisher를 두 개 등록하였다.

```python
self.pub1 = self.create_publisher(String, "message1", 10)
self.pub2 = self.create_publisher(String, "message2", 10)
```

1초마다 하나의 Callback에서 두 메시지를 차례대로 발행한다.

```text
timer_callback()
     ├── message1 발행
     └── message2 발행
```

# 34. 시간을 발행하는 `t_pub`

파일 위치:

```text
khj_example_1/khj_example_1/t_pub.py
```

```python
import rclpy
from rclpy.node import Node
from std_msgs.msg import Header


class Time_pub(Node):
    def __init__(self):
        super().__init__("t_pub")

        self.create_timer(1 / 10, self.timer_callback)

        self.pub = self.create_publisher(
            Header,
            "time",
            10,
        )

    def timer_callback(self):
        msg = Header()

        msg.stamp = self.get_clock().now().to_msg()
        msg.frame_id = "time"

        self.pub.publish(msg)


def main(args=None):
    rclpy.init(args=args)

    node = Time_pub()
    rclpy.spin(node)

    node.destroy_node()
    rclpy.shutdown()


if __name__ == "__main__":
    main()
```

`/t_pub` 노드는 0.1초마다 `/time` 토픽을 발행한다.

```text
노드: /t_pub
토픽: /time
자료형: std_msgs/msg/Header
주기: 0.1초
주파수: 10 Hz
```

# 35. 두 토픽을 구독하는 `mt_sub`

파일 위치:

```text
khj_example_1/khj_example_1/mt_sub.py
```

```python
import rclpy
from rclpy.node import Node
from std_msgs.msg import Header
from std_msgs.msg import String


class Message_time_sub(Node):
    def __init__(self):
        super().__init__("mt_sub")

        self.create_subscription(
            String,
            "message1",
            self.message_callback,
            10,
        )

        self.create_subscription(
            Header,
            "time",
            self.time_callback,
            10,
        )

    def message_callback(self, msg):
        self.get_logger().info(msg.data)

    def time_callback(self, msg):
        self.get_logger().info(
            f"time: {msg.stamp.sec}.{msg.stamp.nanosec}"
        )


def main(args=None):
    rclpy.init(args=args)

    node = Message_time_sub()
    rclpy.spin(node)

    node.destroy_node()
    rclpy.shutdown()


if __name__ == "__main__":
    main()
```

하나의 노드에 두 개의 Subscriber를 등록하였다.

```python
self.create_subscription(
    String,
    "message1",
    self.message_callback,
    10,
)

self.create_subscription(
    Header,
    "time",
    self.time_callback,
    10,
)
```

각 토픽은 서로 다른 Callback 함수에서 처리한다.

```text
/message1 도착
      ↓
message_callback() 실행

/time 도착
      ↓
time_callback() 실행
```

하나의 노드는 서로 다른 자료형의 토픽을 여러 개 구독할 수 있다.
