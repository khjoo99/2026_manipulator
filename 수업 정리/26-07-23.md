# 1. Service Server

`service_server.py`는 두 정수를 입력받아 합을 계산하고, 결과가 홀수인지 판별한다.

사용한 사용자 정의 Service:

```text
user_interface/srv/AddAndOdd
```

Service 구조:

```text
int32 inta
int32 intb
---
int32 sum
string odd
```

Service Server 생성:

```python
self.create_service(
    AddAndOdd,
    "add_server",
    self.add_callback
)
```

요청받은 두 정수의 합을 계산한다.

```python
response.sum = request.inta + request.intb
```

처리 시간이 오래 걸리는 상황을 만들기 위해 10초 동안 대기한다.

```python
time.sleep(10)
```

합이 홀수인지 판별한다.

```python
if response.sum % 2:
    response.odd = "Two ints sum is odd"
else:
    response.odd = "Two ints sum is not odd"
```

계산한 결과를 Client에 반환한다.

```python
return response
```

실행:

```bash
ros2 run khj_basic service_server
```

# 2. 비동기 Service Client

`service_client.py`는 `/add_server`에 두 정수를 비동기로 요청한다.

Service Client 생성:

```python
self.client = self.create_client(
    AddAndOdd,
    "add_server"
)
```

Service Server가 실행될 때까지 기다린다.

```python
while not self.client.wait_for_service(timeout_sec=1.0):
    self.get_logger().info("service not available")
```

3초마다 `send_request()`를 실행한다.

```python
self.create_timer(3, self.send_request)
```

요청값:

```python
self.request.inta = 4
self.request.intb = 8 + self.count
self.count += 1
```

따라서 요청할 때마다 `intb` 값이 1씩 증가한다.

```text
4 + 8
4 + 9
4 + 10
...
```

비동기 방식으로 요청을 보낸다.

```python
self.future = self.client.call_async(
    self.request
)
```

응답이 도착하면 `done_callback()`을 실행한다.

```python
self.future.add_done_callback(
    self.done_callback
)
```

응답 확인:

```python
def done_callback(self, future):
    response = future.result()

    self.get_logger().info(
        f"{response.sum}"
    )
    self.get_logger().info(
        f"{response.odd}"
    )
```

1초마다 실행되는 별도의 Timer도 등록되어 있다.

```python
self.create_timer(1, self.update)
```

```python
def update(self):
    self.get_logger().info("updating!!")
```

`call_async()`를 사용하므로 Service 응답을 기다리는 동안에도 `update()` Callback을 처리할 수 있다.

실행:

```bash
ros2 run khj_basic service_client
```

# 4. 멀티스레드 Service Server

`service_thread_server.py`는 여러 Callback을 처리하기 위해 멀티스레드 Executor를 사용한다.

필요한 모듈:

```python
import threading

from rclpy.callback_groups import ReentrantCallbackGroup
from rclpy.executors import MultiThreadedExecutor
```

여러 Callback의 동시 실행을 허용하는 Callback Group을 생성한다.

```python
self.callback_group = ReentrantCallbackGroup()
```

Service에 Callback Group을 적용한다.

```python
self.create_service(
    AddAndOdd,
    "add_server",
    self.add_callback,
    callback_group=self.callback_group
)
```

최대 4개의 스레드를 사용하는 Executor를 생성한다.

```python
executor = MultiThreadedExecutor(
    num_threads=4
)

executor.add_node(node)
executor.spin()
```

여러 스레드가 같은 부분을 동시에 실행하는 것을 막기 위해 Lock을 생성한다.

```python
self.lock = threading.Lock()
```

Service Callback에서 Lock을 사용한다.

```python
with self.lock:
    response.sum = (
        request.inta
        + request.intb
    )

    time.sleep(10)

    if response.sum % 2:
        response.odd = (
            "Two ints sum is odd"
        )
    else:
        response.odd = (
            "Two ints sum is not odd"
        )
```

현재 코드는 10초 대기까지 Lock 내부에 있으므로, Lock으로 보호된 부분은 한 번에 하나의 Callback만 실행된다.

실행:

```bash
ros2 run khj_basic service_thread_server
```

Client 실행:

```bash
ros2 run khj_basic service_client
```

기본 Server와 멀티스레드 Server는 모두 `/add_server`를 사용하므로 둘을 동시에 실행하지 않는다.

# 5. Action 구조

Action은 시간이 오래 걸리는 작업에 사용하며 다음 세 부분으로 구성된다.

```text
Goal
Feedback
Result
```

이번 실습에서는 Fibonacci Action을 사용한다.

```text
user_interface/action/Fibonacci
```

Action 구조:

```text
int64 step
---
std_msgs/Header header
int64[] seq
---
int64[] temp_seq
```

| 구분 | 필드 | 의미 |
|---|---|---|
| Goal | `step` | Fibonacci 계산 단계 |
| Result | `seq` | 최종 수열 |
| Feedback | `temp_seq` | 계산 중인 수열 |

---

# 6. Action Server

`action_server.py`는 Fibonacci 수열을 계산하여 중간 결과와 최종 결과를 전달한다.

Action Server 생성:

```python
self.action_server = ActionServer(
    self,
    Fibonacci,
    "fibonacci_server",
    execute_callback=self.execute_callback
)
```

Client가 전송한 Goal을 확인한다.

```python
goal = goal_handle.request
step = goal.step
```

Feedback 객체와 수열의 초기값을 생성한다.

```python
feedback_msg = Fibonacci.Feedback()
feedback_msg.temp_seq = [0, 1]
```

Fibonacci 수열을 계산한다.

```python
for i in range(1, step):
    feedback_msg.temp_seq.append(
        feedback_msg.temp_seq[i]
        + feedback_msg.temp_seq[i - 1]
    )
```

한 단계씩 계산할 때마다 Feedback을 보낸다.

```python
goal_handle.publish_feedback(
    feedback_msg
)
```

```python
time.sleep(1)
```

계산이 완료되면 Goal을 성공 상태로 변경한다.

```python
goal_handle.succeed()
```

최종 결과를 생성하여 반환한다.

```python
result = Fibonacci.Result()
result.seq = feedback_msg.temp_seq

return result
```

실행:

```bash
ros2 run khj_basic action_server
```

# 7. Action Client

`action_client.py`는 실행할 때 입력한 숫자를 Goal로 전송한다.

Action Client 생성:

```python
self.action_client = ActionClient(
    self,
    Fibonacci,
    "fibonacci_server"
)
```

실행할 때 입력한 값을 `step`으로 사용한다.

```python
goal_msg = Fibonacci.Goal()
goal_msg.step = int(step)
```

Server가 실행될 때까지 기다린다.

```python
while not self.action_client.wait_for_server(
    timeout_sec=1
):
    self.get_logger().info(
        "fibonacci server is not available!!"
    )
```

Goal을 비동기로 전송한다.

```python
self.future = (
    self.action_client.send_goal_async(
        goal_msg,
        feedback_callback=(
            self.feedback_callback
        )
    )
)
```

Goal 응답 Callback을 등록한다.

```python
self.future.add_done_callback(
    self.goal_response_callback
)
```

Goal 수락 여부 확인:

```python
if not goal_handle.accepted:
    self.get_logger().info(
        "goal rejected!!"
    )
    return

self.get_logger().info(
    "goal accepted!!"
)
```

Goal이 수락되면 최종 결과를 요청한다.

```python
self.get_result_future = (
    goal_handle.get_result_async()
)

self.get_result_future.add_done_callback(
    self.get_result_callback
)
```

Feedback 출력:

```python
def feedback_callback(self, msg):
    feedback = msg.feedback

    self.get_logger().info(
        f"{list(feedback.temp_seq)}"
    )
```

최종 Result 출력:

```python
if result.status == GoalStatus.STATUS_SUCCEEDED:
    self.get_logger().info(
        f"result: {list(result.result.seq)}"
    )
```

중단 또는 취소 상태도 확인한다.

```python
if result.status == GoalStatus.STATUS_ABORTED:
    self.get_logger().info("aborted!!")

if result.status == GoalStatus.STATUS_CANCELED:
    self.get_logger().info("canceled!!")
```

# 9. Service와 Action 비교

| 구분 | Service | Action |
|---|---|---|
| 통신 구조 | Request, Response | Goal, Feedback, Result |
| 사용 목적 | 비교적 짧은 요청 | 시간이 오래 걸리는 작업 |
| 중간 결과 | 없음 | Feedback으로 전달 |
| 이번 실습 | 두 정수 계산 | Fibonacci 수열 계산 |

# 11. 실습 핵심 정리

1. Service는 Request와 Response 방식으로 동작한다.
2. `call_async()`를 사용하면 응답을 기다리는 동안 다른 Callback을 처리할 수 있다.
3. `MultiThreadedExecutor`는 여러 스레드를 이용해 Callback을 처리한다.
4. `ReentrantCallbackGroup`은 같은 그룹의 Callback이 동시에 실행되는 것을 허용한다.
5. `threading.Lock`은 여러 스레드가 같은 부분을 동시에 실행하지 못하도록 한다.
6. Action은 Goal, Feedback, Result로 구성된다.
7. Action Server는 Fibonacci 계산 중간 결과를 Feedback으로 전달한다.
8. Action Client는 Goal 수락 여부, Feedback 및 최종 Result를 비동기로 확인한다.
