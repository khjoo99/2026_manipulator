# 2. turtlesim 실습

## 2.1 GUI 실행 확인

WSL에서 GUI 프로그램이 실행되는지 확인하기 위해 `xeyes`를 설치한다.

```bash
sudo apt install x11-apps
xeyes
```

눈 모양 창이 나타나고 마우스 포인터를 따라 움직이면 GUI 실행 환경이 정상이다.

---

## 2.2 turtlesim 노드 실행

```bash
ros2 run turtlesim turtlesim_node
```

> ROS 2에서 단위가 별도로 명시되지 않은 경우 일반적으로 길이는 `m`, 각도는 `rad`를 사용한다.

---

## 2.3 노드 이름 변경

기본 노드 이름 `/turtlesim`을 `/my_turtle`로 변경하여 실행한다.

```bash
ros2 run turtlesim turtlesim_node \
  --ros-args \
  --remap __node:=my_turtle
```

노드 이름을 변경한 경우 이후 명령어에서 `/turtlesim` 대신 `/my_turtle`을 사용해야 한다.

---

## 2.4 노드 확인

현재 실행 중인 노드 목록을 확인한다.

```bash
ros2 node list
```

`turtlesim` 노드의 상세 정보를 확인한다.

```bash
ros2 node info /turtlesim
```

노드 정보에서는 다음 항목을 확인할 수 있다.

- Subscribers
- Publishers
- Service Servers
- Service Clients
- Action Servers
- Action Clients

---

## 2.5 키보드로 거북이 조종

새로운 터미널에서 다음 명령어를 실행한다.

```bash
ros2 run turtlesim turtle_teleop_key
```

방향키를 이용하여 거북이를 움직일 수 있다.

> 거북이가 바라보는 앞쪽 방향이 거북이 좌표계의 `+x` 방향이다.

---

## 2.6 토픽 확인

현재 사용 중인 토픽 목록을 확인한다.

```bash
ros2 topic list
```

키보드 조종 노드가 발행하는 속도 명령을 확인한다.

```bash
ros2 topic echo /turtle1/cmd_vel
```

`/turtle1/cmd_vel` 토픽은 다음 메시지 형식을 사용한다.

```text
geometry_msgs/msg/Twist
```

---

## 2.7 속도 명령 직접 발행

키보드를 사용하지 않고 `/turtle1/cmd_vel` 토픽을 직접 발행한다.

```bash
ros2 topic pub --rate 1 \
  /turtle1/cmd_vel \
  geometry_msgs/msg/Twist \
  "{linear: {x: 2.0, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 1.8}}"
```

### 명령 값

| 항목 | 값 | 의미 |
|---|---:|---|
| `linear.x` | `2.0` | 앞으로 이동하는 속도 |
| `angular.z` | `1.8` | 반시계 방향 회전 속도 |
| `--rate 1` | `1 Hz` | 1초에 한 번씩 메시지 발행 |

`linear.x`와 `angular.z`가 동시에 적용되므로 거북이는 원을 그리며 움직인다.

---

## 2.8 노드와 토픽 연결 시각화

다음 명령어를 실행한다.

```bash
rqt_graph
```

`rqt_graph`에서 노드와 토픽의 연결 관계를 그래프로 확인할 수 있다.

예시 흐름:

```text
/turtle_teleop → /turtle1/cmd_vel → /turtlesim
```

---

## 2.9 rqt를 이용한 서비스 호출

```bash
rqt
```

상단 메뉴에서 다음 항목을 선택한다.

```text
Plugins → Services → Service Caller
```

Service Caller를 이용하면 ROS 2 서비스를 GUI에서 선택하고 호출할 수 있다.

---

## 2.10 액션 발행

액션 인터페이스의 구조를 확인한다.

```bash
ros2 interface proto turtlesim/action/RotateAbsolute
```

거북이를 절대 각도 `1.57 rad` 방향으로 회전시킨다.

```bash
ros2 action send_goal \
  /turtle1/rotate_absolute \
  turtlesim/action/RotateAbsolute \
  "{theta: 1.57}"
```

`1.57 rad`는 약 `90°`이다.

# 3. ROS 2 파라미터

## 3.1 파라미터 목록 확인

`ros2 node info` 명령어에서는 노드의 파라미터 이름을 직접 확인할 수 없다.

파라미터 목록은 다음 명령어로 확인한다.

```bash
ros2 param list /turtlesim
```

많은 ROS 2 노드에는 다음 파라미터가 공통으로 존재한다.

```text
use_sim_time
```

`use_sim_time`은 실제 시스템 시간 대신 시뮬레이션 시간을 사용할지 결정하는 파라미터이다.

---

## 3.2 배경색 파라미터 확인

배경색은 RGB 값으로 설정한다.

### 파란색 값 확인

```bash
ros2 param get /turtlesim background_b
```

### 빨간색 값 확인

```bash
ros2 param get /turtlesim background_r
```

### 초록색 값 확인

```bash
ros2 param get /turtlesim background_g
```

RGB 값의 범위는 `0~255`이다.

---

## 3.3 배경색 변경

파란색 값을 변경한다.

```bash
ros2 param set /turtlesim background_b 69
```

빨간색 값을 변경한다.

```bash
ros2 param set /turtlesim background_r 69
```

초록색 값을 변경한다.

```bash
ros2 param set /turtlesim background_g 69
```

세 값을 모두 `69`로 설정하면 배경이 어두운 회색 계열로 변경된다.

---

## 3.4 파라미터를 YAML 파일로 저장

현재 `/turtlesim` 노드에 설정된 파라미터를 YAML 파일로 저장한다.

```bash
ros2 param dump /turtlesim > turtlesim.yaml
```

> `dump`는 파라미터를 환경변수로 설정하는 기능이 아니라, 현재 노드의 파라미터를 YAML 파일로 저장하는 기능이다.

저장된 파일을 확인한다.

```bash
cat turtlesim.yaml
```

YAML 파일은 다음과 같은 구조를 가진다.

```yaml
/turtlesim:
  ros__parameters:
    background_b: 69
    background_g: 69
    background_r: 69
    use_sim_time: false
```

프로젝트의 `etc` 폴더에 저장하려면 다음과 같이 경로를 지정한다.

```bash
ros2 param dump /turtlesim \
  > ~/2026_manipulator/etc/turtlesim.yaml
```

---

## 3.5 저장된 파라미터 불러오기

저장한 YAML 파일을 적용하여 turtlesim 노드를 실행한다.

```bash
ros2 run turtlesim turtlesim_node \
  --ros-args \
  --params-file ./turtlesim.yaml
```

프로젝트의 `etc` 폴더에 저장한 파일을 사용할 경우:

```bash
ros2 run turtlesim turtlesim_node \
  --ros-args \
  --params-file ~/2026_manipulator/etc/turtlesim.yaml
```

> 옵션은 `--param-file`이 아니라 `--params-file`이다.  
> 또한 `--ros-args`와 `--params-file` 사이에 공백이 필요하다.
